# マシン識別子の使用{#using-machine-identifiers}

（FMRMS互換性をサポートする要求を除く）すべてのAdobe Access要求には、個別化の際にクライアントに発行されたマシントークンに関する情報が含まれます。 マシントークンには、個別化の際に割り当てられる識別子であるマシンIDが含まれます。 この識別子を使用して、ユーザーがライセンスを要求したか、ドメインに参加したマシンの数をカウントします。

識別子を使用する方法は2つあります。 このメソ `getUniqueId()` ッドは、個別化の際にデバイスに割り当てられた文字列を返します。 文字列をデータベースに格納し、識別子で検索できます。 ただし、この識別子は、ユーザーがハードドライブを再フォーマットし、再度個別化する場合に変更されます。 また、この識別子の値は、同じマシン上の異なるブラウザーのAdobe® AIR®とAdobe® Flash® Playerの間でも異なります。

マシンをより正確にカウントするには、を使用して、識別子 `getBytes()` 全体を保存することができます。 マシンが以前に表示されたことがあるかどうかを確認するには、ユーザー名のすべての識別子を取得し、が呼び出されて一致 `matches()` するかどうかを確認します。 この方法 `matches()``MachineId.getBytes`はによって返される値を比較するために使用する必要があるので、比較する値の数が少ない場合（例えば、特定のユーザーに関連付けられたマシン）にのみ、このオプションは実用的です。
