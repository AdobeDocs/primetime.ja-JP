# プロキシ MVPD SAML 統合

>[!NOTE]
>
>このページのコンテンツは、情報提供の目的でのみ提供されます。 この API を使用するには、Adobeの現在のライセンスが必要です。 不正な使用は許可されていません。

## 概要 {#overview-proxy-mvpd-saml-int}

このドキュメントでは、プロキシ統合の SAML 認証フローについて説明します。  これらのフローは、Adobe Primetime認証サーバー設定に存在するプロキシ設定データに依存します。 プロキシ MVPD は、そのプロキシ設定データをAdobe Primetime認証プロキシ Web サービスを介してAdobe Primetime認証サーバーにプッシュします。

## プロキシ設定データ {#proxy-config-data}

各 MVPD プロキシは、プロキシ化された MVPD のプロキシ設定データをAdobe Primetime認証プロキシ Web サービスに提供します。  詳しくは、プロキシ Web サービスのドキュメントを参照してください。   SAML AuthN フローを機能させるには、プロキシ設定データに次のプロパティを含める必要があります。

| プロパティ | 説明 |
|------------------------|----------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| MVPD ID | 内部的にAdobe Primetime認証にプロキシ化された MVPD を表す文字列。  Adobe Primetime認証のコンテキストでAdobeが一意であることを確認する。 |
| MVPD のデフォルトロゴ URL | ユーザーの MVPD Selector エクスペリエンスに表示できるロゴの URL。  透明な背景を使用する必要があります。 |
| MVPD 表示名 | ロゴと共に表示できる表示名テキストとして使用される文字列。代替テキストとして使用される場合もあります。 |



## SAML 統合フロー {#saml-int-flows}

MVPD 加入者がプログラマのサイトまたはアプリケーションを訪問すると、Adobe Primetime認証は、そのプログラマに対して有効化された MVPD のリストを持つサイトまたはアプリケーションからの API 呼び出しに応答します。  統合は、直接またはプロキシ化できます。プログラマとの間に違いはありません。 これにより、プログラマーは、任意の方法でアクティブな MVPD のリストを表示することができます。 サブスクライバーは MVPD を選択し、Adobe Primetime認証により、サブスクライバーは MVPD の特定の ID プロバイダーにリダイレクトされます。

統合 MVPD プロキシの場合、Adobe Primetime認証と MVPD プロキシの間で統合が行われます。 Adobe Primetime認証は、ユーザ認証要求を MVPD プロキシに送信し、MVPD プロキシがリダイレクトを処理します。 MVPD プロキシがユーザー認証リクエストのリダイレクト先を知るために、Adobe Primetime認証は SAML 認証リクエストで MVPD 識別子を送信します。  この識別子は、上述のように、プロキシ Web サービスを介してプロキシプロバイダーによって指定された MVPD ID です。

### 認証 {#authn-saml-int}

Adobe Primetime認証をプロキシ MVPD と統合するには、以下が必要です。

* プロキシ MVPD が提供する、プロキシプロキシプロキシ Web サービスにプッシュされたプロキシ MVPD のAdobeリスト

* 親 MVPD プロキシの SAML メタデータ

* （推奨） — プロキシ MVPD は、プロキシ MVPD のログインページ URL への追加のリダイレクトを処理します

* MVPD プロキシは、次の IP に対してポート 443 および 80 を開く必要があります。
   * 192.150.4.5
   * 192.150.10.200
   * 192.150.11.4
   * 4.53.93.130
   * 193.105.140.131
   * 193.105.140.132
   * 76.74.170.204
   * 63.140.39.4
   * 66.235.132.38
   * 66.235.139.38
   * 66.235.139.168


#### 認証 SAML リクエストと応答 {#authn-saml-req-resp}

SAML AuthN リクエストでは、プロキシ統合に次の追加プロパティが含まれます。このプロパティは MVPD プロキシで処理する必要があります。  このプロパティは、プロキシ化された MVPD の代わりに要求者を正しく処理し、適切なログインエクスペリエンスをレンダリングするために必要です。 （以下のリクエスト例では、このプロパティが強調表示されています）。

**スコーピングプロパティ**  — 特定の MVPD_ID と MVPD 名を含む IDPEntry 項目を含めます。  これは、ユーザーが実際にプログラマーのピッカーから選択した MVPD を表し、プロキシ Web サービスで指定された MVPD_ID と一致します。

RequestorID には追加のスコーピングプロパティがあります。このプロパティを使用して、プログラマの特定のブランドへのログインをカスタマイズできます（必要に応じて）。 または、単にリクエストの送信元の分析に使用できます。

SAML AuthN 応答では、プロキシ MVPD は次のプロパティで、Proxied MVPD を IdP エンティティとして指定する必要があります。

* SAML 発行者
* 名前修飾子

### 認証 {#authz-proxy-mvpd-saml-int}

認証部分の場合、MVPD は、プログラマーが指定したリソースの認証を受け入れる必要があります。  ほとんどの場合、これは、TBS や TNT など、チャネルネットワークの文字列識別子です。

#### 認証 SAML リクエストと応答 {#authz-saml-req-resp}
